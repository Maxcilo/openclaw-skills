# Twitter 监控配置

## 监控账号

1. **@luyuban69** (鹿鱼班)
2. **@AWUFDC** (AWUFDC)

## 监控目标

发现这些账号发布的**项目相关推文**时，立即通知到Telegram。

## 项目关键词

脚本会自动识别包含以下关键词的推文：
- 中文：项目、代币、空投、铸造、上线、预售、合约地址、官网、白名单
- 英文：project, token, airdrop, mint, launch, presale, IDO, ICO, contract, website, whitelist, OG, WL
- 技术：NFT, DeFi, GameFi, CA:, 0x, discord, telegram

## 使用方法

### 测试通知
```bash
node twitter-monitor.js --test
```

### 单次检查
```bash
node twitter-monitor.js
```

### 持续监控（每5分钟检查一次）
```bash
node twitter-monitor.js --loop
```

### 查看帮助
```bash
node twitter-monitor.js --help
```

## 自动化设置

使用 OpenClaw cron 设置定时任务，每5分钟自动检查：

## 通知格式

```
🐦 推特项目提醒

👤 账号：@luyuban69 (鹿鱼班)
📝 内容：[推文内容]
🔗 链接：[推文链接]
⏰ 时间：[发布时间]
```

## 状态文件

监控状态保存在：`data/twitter-monitor-state.json`

记录每个账号最后检查的推文ID，避免重复通知。

## 注意事项

⚠️ **当前版本限制：**
- 使用浏览器自动化方式获取推文（可能不稳定）
- Twitter 可能有反爬虫机制
- 建议检查间隔不要太频繁（5分钟较合适）

💡 **未来改进：**
- 接入 Twitter API（需要开发者账号）
- 使用 nitter 等第三方服务
- 添加更智能的项目识别（AI分析）
- 支持更多过滤条件

## 监控数据

所有监控记录会保存到日志文件，方便后续分析。
