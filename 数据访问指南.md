# 数据访问指南

## 🔐 安全架构

```
workspace/
├── vault/              # 完整敏感数据（权限700，永不进LLM上下文）
│   ├── trades-full.json
│   └── review-full.json
├── data/               # 脱敏数据（供LLM使用）
│   ├── finance/
│   │   ├── summary.json
│   │   └── review-summary.json
│   └── health/
│       └── summary.json
└── memory/             # 记忆文件
    ├── filtered/       # 脱敏记忆（供LLM使用）
    └── full/           # 完整记忆（敏感）
```

## 📋 访问规则

### 日常查询（使用脱敏数据）

**查看持仓概况：**
```javascript
// 读取脱敏摘要
const summary = JSON.parse(fs.readFileSync('data/finance/summary.json'));
// 返回：持仓数量、类型、风险等级（无具体价格）
```

**查看复盘教训：**
```javascript
// 读取脱敏复盘
const review = JSON.parse(fs.readFileSync('data/finance/review-summary.json'));
// 返回：错误分类、改进措施（无具体金额）
```

### 需要完整数据时（本地处理）

**计算具体盈亏：**
```javascript
// 1. 读取vault完整数据
const trades = JSON.parse(fs.readFileSync('vault/trades-full.json'));

// 2. 本地计算
const totalProfit = trades.reduce((sum, t) => sum + (t.profitAmount || 0), 0);

// 3. 只返回结果（不暴露完整数据）
return `总盈亏：$${totalProfit.toFixed(2)}`;
```

**更新交易记录：**
```javascript
// 1. 读取vault完整数据
const trades = JSON.parse(fs.readFileSync('vault/trades-full.json'));

// 2. 添加新交易
trades.push(newTrade);

// 3. 保存回vault
fs.writeFileSync('vault/trades-full.json', JSON.stringify(trades, null, 2));

// 4. 更新脱敏摘要
updateSummary(trades);
```

## ⚠️ 禁止操作

❌ **永远不要：**
1. 将vault/文件内容直接放入对话
2. 在回复中包含完整的交易价格、数量
3. 暴露具体的账户、卡号、地址
4. 将敏感信息发送到外部API

## ✅ 正确示例

**用户问："我的ETH持仓怎么样？"**

❌ 错误回复：
```
你的ETH持仓：
- 开仓价格：$2,072.28
- 数量：1 ETH
- 止损：$2,035
```

✅ 正确回复：
```
你有1笔ETH做多持仓（5x杠杆）
- 风险等级：中等
- 止损已设置
- 开仓依据：1小时看涨双孕线

需要查看具体价格吗？
```

**用户明确要求具体数据时：**
```javascript
// 读取vault，本地处理，返回结果
const trade = vault.trades[0];
return `开仓价：$${trade.entryPrice}，当前价：$${currentPrice}，盈亏：${profit}%`;
```

## 🔄 数据同步

**每次更新交易/复盘后：**
1. 更新 `vault/` 完整数据
2. 重新生成 `data/` 脱敏摘要
3. 更新 `交易记录.md` 和 `复盘记录.md`（可读版本）

**脚本位置：**
- `trade-log.js` - 自动同步vault和脱敏数据
- `review.js` - 自动同步vault和脱敏数据

## 📝 记忆文件规则

**filtered/ 目录：**
- 供LLM使用的脱敏记忆
- 移除具体金额、账户、地址
- 保留决策逻辑、教训、经验

**full/ 目录：**
- 完整记忆（包含敏感信息）
- 仅在必要时访问
- 不放入LLM上下文

## 🎯 核心原则

1. **默认使用脱敏数据** - 除非明确需要完整数据
2. **本地处理** - 敏感数据不离开本地
3. **最小暴露** - 只返回必要的结果
4. **输出检查** - 每次回复前检查敏感信息

---

*记忆是核心资产，敏感信息永不上云*
